ACME Corporation - Information Security Password Policy
Document Version: 2.1
Effective Date: January 2025
Document Owner: IT Security Team

1. PURPOSE AND SCOPE
This policy establishes password requirements for all ACME Corporation systems, applications, and network resources. This policy applies to all employees, contractors, and third-party users with access to ACME systems.

2. PASSWORD REQUIREMENTS

2.1 Complexity Requirements
All passwords must meet the following minimum criteria:
- Minimum length: 12 characters
- Must contain at least one uppercase letter (A-Z)
- Must contain at least one lowercase letter (a-z)
- Must contain at least one number (0-9)
- Must contain at least one special character (!@#$%^&*)
- Cannot contain dictionary words or common phrases
- Cannot contain user's name, username, or company name
- Cannot reuse the last 5 passwords

2.2 Password Rotation
- Standard user accounts: Password must be changed every 90 days
- Administrator accounts: Password must be changed every 60 days
- Service accounts: Password must be changed every 180 days with IT approval
- System will provide reminder notifications 7 days before expiration

2.3 Account Lockout Policy
- After 5 consecutive failed login attempts, the account will be locked
- Lockout duration: 30 minutes
- After 3 lockouts in 24 hours, manual administrator intervention required
- Security team will be notified of repeated lockout attempts

3. PASSWORD STORAGE AND TRANSMISSION
- Passwords must never be stored in plain text
- All passwords must be encrypted using industry-standard algorithms (AES-256 or better)
- Passwords must never be transmitted via email or instant messaging
- Multi-factor authentication (MFA) is required for remote access

4. PASSWORD SHARING AND DISCLOSURE
- Passwords are confidential and must not be shared with anyone
- Each user must have a unique password
- Shared or group accounts are prohibited unless approved by IT Security
- Violation of password sharing policy may result in disciplinary action

5. DEFAULT AND TEMPORARY PASSWORDS
- All default passwords must be changed upon first login
- Temporary passwords expire after 24 hours or first use
- Users must create a new password meeting complexity requirements

6. PASSWORD RECOVERY
- Password reset requests must be verified through multi-factor authentication
- Self-service password reset available through company portal
- Help desk verification required for phone-based password resets

7. MULTI-FACTOR AUTHENTICATION (MFA)
- MFA is mandatory for:
  * All remote access (VPN, cloud applications)
  * Administrator and privileged accounts
  * Access to sensitive data systems
  * Email systems accessed from external networks

8. MONITORING AND COMPLIANCE
- IT Security will conduct regular audits of password compliance
- Automated scanning for weak or compromised passwords
- Quarterly reviews of access logs and authentication failures
- Annual security awareness training required for all users

9. EXCEPTIONS
Any exceptions to this policy must be:
- Documented in writing
- Approved by the Chief Information Security Officer (CISO)
- Reviewed annually
- Compensating controls must be implemented

10. POLICY VIOLATIONS
Violations of this password policy may result in:
- Account suspension or termination
- Disciplinary action up to and including termination of employment
- Legal action if malicious intent is determined

11. POLICY REVIEW
This policy will be reviewed and updated:
- Annually
- After any security incident involving password compromise
- When technology or business requirements change

For questions or clarifications regarding this policy, contact the IT Security Team at security@acme.com.
